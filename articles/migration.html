    Migration To 1.0.0

    Version 1.0.0 is a major update. According to the package changelog and current source, it adds Android support, HTTPS NTRIP options, explicit NTRIP callbacks, structured GPS disconnect notifications, and binary distribution changes.

    Main Breaking Changes

    Before 1.0.0 In 1.0.0
    Interfaces without the I prefix IBluetoothService, IPeripheralService, IGPSService, INtripService
    lowerCamelCase methods and properties PascalCase API
    Vigram.Init(token) Vigram.Initial(token)
    Vigram.bluetoothService Vigram.BluetoothService
    bluetooth.startScan() bluetooth.StartScan()
    bluetooth.connect(peripheral) bluetooth.Connect(peripheral)
    Vigram.peripheralService(peripheral) Vigram.PeripheralService(peripheral)
    peripheral.start() peripheral.Start()
    Method-style NMEA callbacks Assign IPeripheralService.Nmea
    Vigram.gpsService(peripheral, ntripTask) Vigram.GPSService(peripheral, bluetoothService, ntripTask, automaticallyReconnectNtrip)
    NTRIP Start() without handshake result Start(actionSuccess, actionFailure)
    NTRIP data callback without an error channel Data(Action<byte[], Exception>)
    Separate GGAMessage, GSTMessage, TXTMessage types NmeaMessage.Gga, NmeaMessage.Gst, NmeaMessage.Txt
    NAV-DOP/NAV-PVT methods with callbacks ChangeStatusNAVDOP(bool) and ChangeStatusNAVPVT(bool)

    Renaming Example

    // 1.0.0
IAuthenticationService authentication = Vigram.Initial("YOUR_SDK_TOKEN");
IBluetoothService bluetooth = Vigram.BluetoothService;
IPeripheralService peripheral = Vigram.PeripheralService(selectedPeripheral);
INtripService ntrip = Vigram.NtripService;
ILaserService laser = Vigram.LaserService(peripheral);

    Create these services once for the owning session and pass the instances around. Vigram.BluetoothService, Vigram.NtripService, Vigram.LaserService(...), and the other service factories do not return shared singletons.

    NMEA Handling

    peripheral.Nmea = message =>
{
    if (message is NmeaMessage.Gga gga)
    {
        ntripTask.SetGga(gga);
    }
};

    GPS Service Creation

    IGPSService now needs the Bluetooth service as well as the peripheral and NTRIP task.

    IGPSService gps = Vigram.GPSService(
    peripheral,
    bluetooth,
    ntripTask,
    automaticallyReconnectNtrip: true);

    NTRIP Error Handling

    Always wire both startup callbacks and the correction data error channel.

    ntripTask.Data((bytes, error) =>
{
    if (error != null)
    {
        Debug.LogError(error.Message);
        return;
    }

    Debug.Log(bytes.Length);
});

ntripTask.Start(
    actionSuccess: () => Debug.Log("NTRIP connected"),
    actionFailure: error => Debug.LogError(error.Message));

    GPS Disconnect Reasons

    IGPSService.ShouldDisconnect reports:

    • ConfigurationFailed;
    • DeviceNotFound;
    • NtripDisrupted;
    • InvalidGnssPosition.

    Use this callback for user-facing reconnect UI, diagnostics, and support logs.

    What Not To Migrate Into Public App Code

    Do not migrate old examples that used internal protocol classes, raw authentication structures, or test-only software update APIs. In 1.0.0, application integrations should use public service interfaces and typed callbacks.
