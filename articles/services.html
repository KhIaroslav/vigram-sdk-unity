    Core Services

    Vigram

    Vigram is the public static factory surface. Most members below create new service objects; they are not singleton accessors. Store returned services in fields owned by your session, screen, or coordinator and pass those same instances to dependent components.

    Factory Returns Purpose
    Initial(string token) IAuthenticationService Validate SDK access
    TokenIsValid AuthenticationResult Read the cached authentication result
    BluetoothService IBluetoothService Bluetooth LE scan, connect, and disconnect
    PeripheralService(...) IPeripheralService viDoc protocol, messages, requests, and configuration
    NtripService INtripService Mountpoint discovery and NTRIP task creation
    GPSService(...) IGPSService GPS aggregation, NTRIP state, correction state, reconnect signals
    LaserService(IPeripheralService) ILaserService Laser state and distance measurements
    DeviceService() IDeviceService Unity device motion updates
    LoggerService(string filename) ILogger File logging for SDK events

    Recommended lifetime model:

    Service Typical owner
    IAuthenticationService Startup/authentication flow
    IBluetoothService One active viDoc session or device picker screen
    IPeripheralService One selected BPeripheral session
    INtripService Session object or NTRIP configuration screen
    INtripTaskService One selected NTRIP mountpoint/correction stream
    IGPSService One corrected GPS session
    ILaserService One peripheral session that exposes laser UI
    IDeviceService One motion-observation owner
    ILogger One logged session/file

    IAuthenticationService

    Use Vigram.Initial(token).Check(...) before Bluetooth, peripheral, NTRIP, GPS, or laser work.

    Vigram.Initial("YOUR_SDK_TOKEN").Check(result =>
{
    if (result is AuthenticationResult.Success)
    {
        Debug.Log("Authorized");
    }

    if (result is AuthenticationResult.Error error)
    {
        Debug.LogError($"Auth failed: {error.Enum} {error.Message}");
    }
});

    Failure values are represented by AuthenticationError.

    IBluetoothService

    Owns Bluetooth state, scanning, available BPeripheral updates, connection, and disconnection.

    Member Use
    State(...) Observe platform Bluetooth availability
    ObserveAvailableDevices(...) Receive the current viDoc candidate list
    StartScan() Start scanning
    StopScan() Stop scanning and clear available devices
    Connect(BPeripheral) Connect to the selected device
    Disconnect() Disconnect the active device

    Create one IBluetoothService for a viDoc session and reuse it for scanning, connecting, GPS service creation, and cleanup. Calling Vigram.BluetoothService again creates another service instance.

    BPeripheral

    BPeripheral identifies a discovered viDoc device.

    Member Meaning
    Name Display name received from Bluetooth discovery
    SerialNumber Serial-like value derived from the device name

    IPeripheralService

    Main viDoc protocol service. It receives parsed NMEA, device, and satellite messages and sends high-level requests to the connected device.

    Important callbacks:

    Callback Use
    Nmea Receive NmeaMessage.Gga, NmeaMessage.Gst, and NmeaMessage.Txt
    CurrentDevice Receive parsed viDoc model/capability information
    DeviceMessages Receive battery, version, measurement, IMU, laser state, and other device messages
    SatelliteMessages Receive GNSS configuration and satellite responses
    State Observe connected/disconnected peripheral state
    ConfigurationState Observe startup configuration progress or failure
    ViDocState Receive viDoc warning/error/notice/user state messages
    ViDocResetState Observe reset progress
    PpkMeasurementsState Observe PPK measurement state
    Debug Receive SDK diagnostic messages during integration

    Important methods:

    Method Use
    Start() Initialize the connected peripheral session
    RequestVersion(...) Request hardware/software version
    RequestBattery(...) Request battery percentage
    RequestIMUAngle(...) and related IMU methods Request IMU readings
    GetCurrentStatusGNSS(...) Read enabled state for a navigation system
    ChangeStatusGNSS(...) Enable or disable a navigation system except GPS
    ActivateAllConstellationGNSS() Enable all supported constellations
    ChangeStatusNAVDOP(...) Enable or disable NAV-DOP messages
    ChangeStatusNAVPVT(...) Enable or disable NAV-PVT messages
    GetCurrentMinimumElevation(...) Read the current elevation mask
    SetMinimumElevation(...) Set the elevation mask
    GetChangingRateOfMessages(...) Read GNSS message rate
    SetChangingRateOfMessages(...) Set GNSS message rate
    SetDynamicState(...) Set pedestrian or stationary dynamic model
    GetCurrentDynamicState(...) Read the active dynamic model
    ResetViDoc() Request viDoc reset

    INtripService And INtripTaskService

    INtripService fetches mountpoints and creates INtripTaskService instances. Keep one INtripService for the NTRIP flow and keep the returned task for one correction stream.

    Type Responsibility
    INtripService.Mountpoints(...) Load available mountpoints from a caster
    INtripService.Task(...) Create a correction task for a mountpoint and current GGA
    INtripTaskService.Start(...) Start the correction connection
    INtripTaskService.Data(...) Receive correction bytes or terminal stream errors
    INtripTaskService.SetGga(...) Update the task with the current position
    INtripTaskService.Disconnect() Stop the correction stream

    See Fetch NTRIP Mountpoints and Start An NTRIP Task.

    IGPSService

    Combines IPeripheralService, IBluetoothService, and INtripTaskService.

    Callback/method Use
    Coordinate(...) Current coordinate
    TimestampedCoordinate(...) Current coordinate with timestamp
    HorizontalAccuracy(...) / VerticalAccuracy(...) Accuracy values in meters
    Quality(...) GPS quality indicator
    Hdop(...) Horizontal dilution of precision
    IsCorrecting(...) Whether corrected data is active
    ShouldDisconnect(...) Structured disconnect/reconnect signal
    Start(...) Start NTRIP/GPS processing
    Reconnect() Attempt reconnect

    See Start GPS With Automatic NTRIP Reconnect.

    ILaserService

    Created on top of an active IPeripheralService.

    Method Use
    GetLasersStatus(...) Read current laser state
    TurnLaserOn(...) Turn on the selected laser position
    TurnLaserOff(...) Turn off the selected laser position
    Record(...) Start a measurement session

    Keep one laser service for the peripheral session. For devices without lasers, hide or disable laser features based on CurrentDevice.

    See Record A Laser Measurement.

    IDeviceService

    Exposes mobile device motion updates and is independent from the viDoc peripheral session.

    IDeviceService device = Vigram.DeviceService();
device.StartDeviceMotionUpdates(motion => Debug.Log(motion.Attitude));
device.StopDeviceMotionUpdates();

    Keep the same IDeviceService instance so StopDeviceMotionUpdates() stops the updates that were started by that owner.

    ILogger

    Vigram.LoggerService(filename) creates an SDK logger. Attach it before IPeripheralService.Start() so startup, configuration, NMEA, device, and satellite events are captured from the beginning of the session.

    If the path is relative, the SDK writes under Application.persistentDataPath. Android also supports the android-documents:// scheme for the Documents directory.

    See Log A Full Session.
