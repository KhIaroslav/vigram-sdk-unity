    Platforms And Build

    iOS

    The package includes native iOS plugins:

    • Runtime/Plugins/iOS/BluetoothLE;
    • Runtime/Plugins/iOS/MotionManager.

    The editor post-build processor Editor/PostBuildProcess.cs adds this key to the generated Info.plist:

    NSBluetoothAlwaysUsageDescription = Needed to work with viGram

    Check the final Info.plist after Xcode export, especially if the host application already provides its own Bluetooth permission text. Production applications should use user-facing permission text that matches the app experience.

    Android

    The Android plugin is located at:

    Runtime/Plugins/Android/BluetoothLE.androidlib

    The package AndroidManifest.xml declares:

    • android.hardware.bluetooth_le;
    • android.permission.BLUETOOTH up to API 30;
    • android.permission.BLUETOOTH_ADMIN up to API 30;
    • android.permission.BLUETOOTH_ADVERTISE;
    • android.permission.BLUETOOTH_CONNECT;
    • android.permission.BLUETOOTH_SCAN;
    • android.permission.ACCESS_COARSE_LOCATION;
    • android.permission.ACCESS_FINE_LOCATION;
    • android.permission.WRITE_EXTERNAL_STORAGE up to API 28.

    The androidlib build.gradle uses compileSdk 36, targetSdk 31, and minSdk 26.

    Android 12 and newer require runtime Bluetooth permissions for scan/connect operations. The host Unity application should request permissions before starting a scan and should handle denial with user-facing UI.

    Render Pipeline

    The SDK does not include rendering components, shaders, scenes, or visual prefabs. It is independent of Built-in Render Pipeline, URP, and HDRP.

    WebGL

    WebGL is not listed in package.json, README.md, or CHANGELOG.md as a supported target. The current package depends on mobile Bluetooth LE, Android/iOS native plugins, and a mobile motion bridge.

    Build Validation Checklist

    1. Install the package through Unity Package Manager.
    2. Select an iOS or Android build target.
    3. Verify platform permissions and generated platform metadata.
    4. Build and run on a physical device.
    5. Check IBluetoothService.State.
    6. Confirm that ObserveAvailableDevices receives viDoc devices.
    7. Connect to a selected BPeripheral.
    8. Subscribe to IPeripheralService.ConfigurationState before Start().
    9. Confirm that Nmea receives GGA/GST messages.
    10. If NTRIP is used, verify mountpoint loading, task startup, and correction state.
    11. If laser features are used, confirm device capabilities through CurrentDevice.
