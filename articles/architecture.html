    Service Lifetime

    Vigram is a static factory surface, not a singleton container. Create service instances at the start of the owning session or screen and pass those instances to other components. The detailed ownership table is in Core Services.

    Runtime Lifecycle

    1. Authenticate the SDK with Vigram.Initial(token).Check(...).
    2. Create one IBluetoothService for the viDoc session.
    3. Subscribe to State and ObserveAvailableDevices.
    4. Start scanning with StartScan().
    5. Let the application choose a BPeripheral.
    6. Call IBluetoothService.Connect(peripheral) on the same Bluetooth service instance.
    7. Create an IPeripheralService.
    8. Subscribe to state, NMEA, device, satellite, and diagnostic callbacks.
    9. Call IPeripheralService.Start().
    10. Create and store INtripService, INtripTaskService, IGPSService, and ILaserService only when the application needs correction streams, GPS aggregation, or laser measurements.
    11. Stop scans, disconnect Bluetooth, stop NTRIP tasks, and stop motion updates on the same service instances when the owning Unity object is destroyed or the user ends the session.

    Scene And Inspector Model

    The package does not provide Unity scenes, prefabs, or ScriptableObject assets. Services are created through Vigram factories from C# code. Documentation therefore focuses on runtime integration patterns rather than Inspector fields.
