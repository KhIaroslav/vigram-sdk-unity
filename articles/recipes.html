    Practical Recipes

    This section turns the public API into integration-oriented workflows. Each recipe focuses on a task that normally appears in a production Unity app.

    Own A Peripheral Session

    Use this pattern after the application has authenticated, scanned, and selected a BPeripheral. The complete first-run flow lives in Installation And First Run; this recipe focuses on keeping service ownership explicit.

    using UnityEngine;
using ViGram;

public class VidocPeripheralSession
{
    private readonly IBluetoothService bluetooth;
    private readonly BPeripheral selectedPeripheral;
    private readonly IPeripheralService peripheral;

    public VidocPeripheralSession(
        IBluetoothService bluetooth,
        BPeripheral selectedPeripheral,
        ILogger logger = null)
    {
        this.bluetooth = bluetooth;
        this.selectedPeripheral = selectedPeripheral;
        peripheral = Vigram.PeripheralService(selectedPeripheral, logger: logger);
    }

    public IPeripheralService Peripheral => peripheral;

    public void Start()
    {
        bluetooth.StopScan();
        bluetooth.Connect(selectedPeripheral);

        peripheral.State = state => Debug.Log($"Peripheral: {state}");
        peripheral.ConfigurationState = state => Debug.Log($"Configuration: {state.GetType().Name}");
        peripheral.Nmea = message => Debug.Log(message.Raw);
        peripheral.Start();
    }

    public void Stop()
    {
        bluetooth.Disconnect();
    }
}

    The session receives the existing IBluetoothService; it does not call Vigram.BluetoothService again. Higher-level UI code can own this class and call Stop() when the session ends.

    Subscribe To NMEA Messages

    Use IPeripheralService.Nmea when the app needs raw NMEA sentences, GGA positions, GST accuracy, or TXT diagnostics. For best SDK behavior, configure the viDoc device to emit NMEA-0183 GGA and GST messages.

    using UnityEngine;
using ViGram;

public class NmeaConsumer
{
    private NmeaMessage.Gga latestGga;

    public NmeaMessage.Gga LatestGga => latestGga;

    public void Attach(IPeripheralService peripheral)
    {
        peripheral.Nmea = message =>
        {
            switch (message)
            {
                case NmeaMessage.Gga gga:
                    latestGga = gga;
                    Debug.Log($"GGA: {gga.Coordinate?.Latitude}, {gga.Coordinate?.Longitude}, quality: {gga.Quality}");
                    break;

                case NmeaMessage.Gst gst:
                    Debug.Log($"GST accuracy: H={gst.Accuracy.Horizontal} m, V={gst.Accuracy.Vertical} m");
                    break;

                case NmeaMessage.Txt txt:
                    Debug.Log($"TXT: {txt.Message}");
                    break;
            }
        };
    }
}

    Keep the latest NmeaMessage.Gga if the app will create an NTRIP task. NTRIP needs a current position so the caster can return relevant corrections.

    Read Device Version, Battery, And Capabilities

    Use this recipe after IPeripheralService.Start() to populate diagnostics UI and to decide whether laser features should be shown.

    using UnityEngine;
using ViGram;

public class DeviceInfoReader
{
    public void Attach(IPeripheralService peripheral)
    {
        peripheral.CurrentDevice = device =>
        {
            Debug.Log($"Device type: {device.TypeOfDevice}");
            Debug.Log($"Front laser: {device.HasFrontLaser}");
            Debug.Log($"Bottom laser: {device.HasBottomLaser}");
            Debug.Log($"Housing: {device.GetHousing}");
            Debug.Log($"Mount: {device.GetMount}");

            if (device.TypeOfDevice == DeviceVersion.ViDocLight)
            {
                Debug.Log("Disable laser controls for viDoc Light.");
            }

            if (device.TypeOfDevice == DeviceVersion.ViDocWithOldSoftware)
            {
                Debug.LogWarning("Device software is old. Ask the user to update firmware.");
            }

            if (device.TypeOfDevice == DeviceVersion.Unknown)
            {
                Debug.LogError("Unsupported viDoc device.");
            }
        };

        peripheral.RequestBattery(battery =>
            Debug.Log($"Battery: {battery.Percentage}%"));

        peripheral.RequestVersion(version =>
            Debug.Log($"Hardware: {version.Hardware}, software: {version.Software}"));
    }
}

    The exact UI decision is application-specific. A common pattern is to hide laser buttons for ViDocLight, show a firmware update warning for old software, and collect logs for unknown devices.

    Handle Device And Satellite Messages

    Use DeviceMessages when you want one callback for all device-level responses, including measurements, battery, version, IMU, and laser state.

    using UnityEngine;
using ViGram;

public class MessageRouter
{
    public void Attach(IPeripheralService peripheral)
    {
        peripheral.DeviceMessages = message =>
        {
            switch (message)
            {
                case DeviceMessage.Battery battery:
                    Debug.Log($"Battery: {battery.Percentage}%");
                    break;

                case DeviceMessage.Version version:
                    Debug.Log($"Version: {version.Software}");
                    break;

                case DeviceMessage.Measurement measurement:
                    Debug.Log($"Distance: {measurement.Distance} mm, quality: {measurement.Quality}");
                    break;

                case DeviceMessage.LaserState laserState:
                    Debug.Log($"Laser state: {laserState.State}");
                    break;
            }
        };

        peripheral.SatelliteMessages = message =>
        {
            switch (message)
            {
                case SatelliteMessage.Dop dop:
                    Debug.Log($"HDOP: {dop.HorizontalDop}, VDOP: {dop.VerticalDop}");
                    break;

                case SatelliteMessage.Pvt pvt:
                    Debug.Log($"Satellites used: {pvt.SatelliteCount}");
                    break;

                case SatelliteMessage.Acknowledge acknowledge:
                    Debug.Log($"GNSS request acknowledged: {acknowledge.Result}");
                    break;
            }
        };
    }
}

    Use typed request callbacks when the app asks for one specific value. Use broad message callbacks when the app needs to monitor the whole session.

    Configure GNSS Constellations

    Use these methods to control which navigation systems are enabled. GPS is treated as the base system and should not be disabled through ChangeStatusGNSS.

    using UnityEngine;
using ViGram;

public class GnssConstellationConfigurator
{
    public void Configure(IPeripheralService peripheral)
    {
        peripheral.ActivateAllConstellationGNSS();

        peripheral.GetCurrentStatusGNSS(
            NavigationSystemValue.NavigationSystemType.GALILEO,
            status => Debug.Log($"GALILEO enabled: {status.IsEnabled}"));

        peripheral.ChangeStatusGNSS(
            NavigationSystemValue.NavigationSystemType.GLONASS,
            activate: true);

        peripheral.ChangeStatusGNSS(
            NavigationSystemValue.NavigationSystemType.BEIDOU,
            activate: true);
    }
}

    Use this in a settings or diagnostics screen. Avoid changing GNSS configuration repeatedly during active measurement unless the product explicitly needs it.

    Configure NAV-DOP, NAV-PVT, Elevation, Rate, And Dynamic State

    The SDK exposes typed helpers for common viDoc GNSS configuration.

    using UnityEngine;
using ViGram;

public class GnssMessageConfigurator
{
    public void Configure(IPeripheralService peripheral)
    {
        peripheral.ChangeStatusNAVDOP(activate: true);
        peripheral.ChangeStatusNAVPVT(activate: true);

        peripheral.SetMinimumElevation(ElevationValue.Elevation.Angle10);
        peripheral.GetCurrentMinimumElevation(elevation =>
            Debug.Log($"Elevation mask: {elevation.Value}"));

        peripheral.SetChangingRateOfMessages(RateValue.Rate.Hertz7);
        peripheral.GetChangingRateOfMessages(rate =>
            Debug.Log($"Message rate: {rate.Value}"));

        peripheral.SetDynamicState(DynamicStateType.Pedestrian);
        peripheral.GetCurrentDynamicState(state =>
            Debug.Log($"Dynamic state: {state.Value}"));
    }
}

    Recommended defaults depend on the app domain. Walking capture workflows usually use DynamicStateType.Pedestrian; fixed installations can use DynamicStateType.Stationary.

    Fetch NTRIP Mountpoints

    Use INtripService.Mountpoints to present available caster mountpoints to the user or to select one automatically.

    using System.Collections.Generic;
using System.Linq;
using UnityEngine;
using ViGram;

public class NtripMountpointLoader
{
    private readonly INtripService ntrip;

    public NtripMountpointLoader(INtripService ntrip)
    {
        this.ntrip = ntrip;
    }

    public void Load()
    {
        NtripConnectionInformation connection = new(
            hostname: "caster.example.com",
            port: 2101,
            username: "user",
            password: "password",
            forceHttpsConnection: true,
            forceHttpsMountpointsConnection: true);

        ntrip.Mountpoints(
            connection,
            actionSuccess: mountpoints => OnMountpoints(connection, mountpoints),
            actionFailure: error => Debug.LogError($"Mountpoint request failed: {error.Message}"));
    }

    private void OnMountpoints(
        NtripConnectionInformation connection,
        List<NtripMountPoint> mountpoints)
    {
        IEnumerable<NtripMountPoint> rtcmMountpoints = mountpoints
            .Where(point => point.Format.IsRTCM && point.RequiresNMEA);

        foreach (NtripMountPoint point in rtcmMountpoints)
        {
            Debug.Log($"{point.Name} {point.CountryCode} {point.NetworkName}");
        }
    }
}

    Create INtripService once for the NTRIP flow and pass it into helpers like this loader. The viDoc correction flow expects mountpoints that accept NMEA input. Filtering by RequiresNMEA and RTCM-like data formats is a useful first pass, but the final selection still depends on provider rules and field requirements.

    INtripService ntrip = Vigram.NtripService;
NtripMountpointLoader loader = new NtripMountpointLoader(ntrip);
NtripRunner runner = new NtripRunner(ntrip);

    Start An NTRIP Task

    Create the NTRIP task after you have a selected mountpoint and a recent NmeaMessage.Gga.

    using UnityEngine;
using ViGram;

public class NtripRunner
{
    private readonly INtripService ntrip;

    public NtripRunner(INtripService ntrip)
    {
        this.ntrip = ntrip;
    }

    public INtripTaskService CreateAndStart(
        NtripConnectionInformation connection,
        string mountPoint,
        NmeaMessage.Gga latestGga)
    {
        INtripTaskService task = ntrip.Task(connection, mountPoint, latestGga);

        task.NtripState(state => Debug.Log(state.Description()));

        task.Data((bytes, error) =>
        {
            if (error != null)
            {
                Debug.LogError($"Correction stream failed: {error.Message}");
                return;
            }

            Debug.Log($"Correction bytes: {bytes.Length}");
        });

        task.Start(
            actionSuccess: () => Debug.Log("NTRIP connected"),
            actionFailure: error => Debug.LogError($"NTRIP connection failed: {error.Message}"));

        return task;
    }

    public void UpdatePosition(INtripTaskService task, NmeaMessage.Gga latestGga)
    {
        task.SetGga(latestGga);
    }
}

    Keep the returned INtripTaskService for the active correction stream. Call Disconnect() on that same instance when the session ends. If the app moves significantly, keep the NTRIP task updated with fresh GGA messages.

    Start GPS With Automatic NTRIP Reconnect

    Use IGPSService when you want one service to combine corrected location data, NTRIP state, correction status, and disconnect signals.

    using UnityEngine;
using ViGram;

public class GpsRunner
{
    private IGPSService gps;

    public void StartGps(
        IPeripheralService peripheral,
        IBluetoothService bluetooth,
        INtripTaskService ntripTask)
    {
        gps = Vigram.GPSService(
            peripheral,
            bluetooth,
            ntripTask,
            automaticallyReconnectNtrip: true);

        gps.Coordinate(coordinate =>
        {
            if (coordinate.HasValue)
            {
                Debug.Log($"{coordinate.Value.Latitude}, {coordinate.Value.Longitude}");
            }
        });

        gps.TimestampedCoordinate((timestamp, coordinate) =>
            Debug.Log($"{timestamp:o}: {coordinate.Latitude}, {coordinate.Longitude}"));

        gps.HorizontalAccuracy(value => Debug.Log($"Horizontal accuracy: {value} m"));
        gps.VerticalAccuracy(value => Debug.Log($"Vertical accuracy: {value} m"));
        gps.Quality(quality => Debug.Log($"GPS quality: {quality}"));
        gps.Hdop(hdop => Debug.Log($"HDOP: {hdop}"));
        gps.IsCorrecting(isCorrecting => Debug.Log($"Correcting: {isCorrecting}"));

        gps.ShouldDisconnect(signal =>
            Debug.LogWarning($"GPS disconnect reason: {signal.DisconnectReason}"));

        gps.Start(state => Debug.Log(state.Description()));
    }
}

    Keep the IGPSService instance for the active GPS session. Use ShouldDisconnect to switch the UI into a recoverable state. For example, show a reconnect button for NtripDisrupted, ask the user to move into better GNSS conditions for InvalidGnssPosition, or restart discovery for DeviceNotFound.

    Record A Laser Measurement

    Use ILaserService.Record for a measurement session. Use device capabilities to hide laser UI on viDoc Light.

    using UnityEngine;
using ViGram;

public class LaserMeasurementRunner
{
    private readonly ILaserService laser;

    public LaserMeasurementRunner(IPeripheralService peripheral)
    {
        laser = Vigram.LaserService(peripheral);
    }

    public void Measure()
    {
        laser.GetLasersStatus(state =>
            Debug.Log($"Laser status: {state.State}"));

        LaserConfiguration configuration = new(
            shotMode: LaserConfiguration.ShotMode.Auto,
            position: LaserConfiguration.Position.Bottom,
            duration: 10);

        laser.Record(
            configuration,
            actionSuccess: measurement =>
            {
                Debug.Log($"Distance: {measurement.Distance} mm");
                Debug.Log($"Quality: {measurement.Quality} ({measurement.QualityRaw})");
            },
            actionFailure: error => Debug.LogError($"Laser measurement failed: {error.Message}"));
    }
}

    Create one ILaserService for the peripheral session and reuse it for status, on/off commands, and recording. For protocol version 1.0, laser duration is clamped to the supported range of 5 to 60 seconds.

    Turn A Laser On Or Off

    Use explicit laser state control only when your UI needs it. Measurement sessions can often rely on Record.

    // Reuse the ILaserService that belongs to this peripheral session.
laser.TurnLaserOn(LaserConfiguration.Position.Bottom);
laser.TurnLaserOff(LaserConfiguration.Position.Bottom);

    Front laser controls should be shown only when CurrentDevice reports HasFrontLaser.

    Request IMU Data

    Use IMU requests for diagnostics or orientation-related workflows.

    using UnityEngine;
using ViGram;

public class ImuReader
{
    public void Read(IPeripheralService peripheral)
    {
        peripheral.RequestIMUAngle(angle =>
            Debug.Log($"Nick: {angle.NickAngle}, roll: {angle.RollAngle}"));

        peripheral.RequestIMURotation(rotation =>
            Debug.Log($"Nick rate: {rotation.NickRate}, roll rate: {rotation.RollRate}, yaw rate: {rotation.YawRate}"));

        peripheral.RequestIMUCalibrationStatus(status =>
            Debug.Log($"IMU calibration count: {status.Times}"));
    }
}

    Only expose calibration actions in UI if the product has a clear calibration workflow for the user.

    Log A Full Session

    Use a logger when debugging field issues, validating NTRIP behavior, or collecting support data. Create the logger once before creating the logged peripheral service.

    using System.IO;
using UnityEngine;
using ViGram;

public class LoggedSessionFactory
{
    public IPeripheralService Create(BPeripheral selectedPeripheral)
    {
        string file = Path.Combine(Application.persistentDataPath, "vidoc-session.csv");
        ILogger logger = Vigram.LoggerService(file);

        return Vigram.PeripheralService(selectedPeripheral, logger: logger);
    }
}

    Attach the logger before Start() to capture startup and configuration messages. Treat logs as support data: define retention, upload, and deletion behavior in the host application.
