    Troubleshooting

    Authentication Fails

    Check the AuthenticationResult.Error value and message.

    Vigram.Initial("YOUR_SDK_TOKEN").Check(result =>
{
    if (result is AuthenticationResult.Error error)
    {
        Debug.LogError($"{error.Enum}: {error.Message}");
    }
});

    Common causes:

    • the token is missing;
    • the token does not match the application bundle/package;
    • the SDK was deactivated;
    • offline authorization is missing or expired;
    • the device has no connectivity when a fresh online check is required.

    Bluetooth Scan Finds No Devices

    Check:

    • IBluetoothService.State;
    • Android runtime Bluetooth and location permissions;
    • the generated iOS NSBluetoothAlwaysUsageDescription;
    • that the app is running on a physical iOS or Android device;
    • that the viDoc device is powered on and close enough;
    • that scanning is not stopped immediately by app lifecycle code;
    • that the UI is using the latest list from ObserveAvailableDevices.
    bluetooth.State(state => Debug.Log($"Bluetooth state: {state}"));
bluetooth.ObserveAvailableDevices(devices => Debug.Log($"Device count: {devices.Count}"));
bluetooth.StartScan();

    Peripheral Does Not Configure

    Subscribe to IPeripheralService.ConfigurationState, State, and Debug before calling Start().

    peripheral.State = state => Debug.Log($"Peripheral: {state}");
peripheral.ConfigurationState = state => Debug.Log($"Configuration: {state.GetType().Name}");
peripheral.Debug = message => Debug.Log(message);
peripheral.Start();

    Typical causes include Bluetooth disconnects, unsupported device version, old firmware, denied access, write timeouts, or configuration values that are not supported by the connected device.

    No NMEA GGA Or GST Messages Arrive

    Check:

    • IPeripheralService.Start() was called after Bluetooth connect;
    • Nmea was assigned before or immediately after startup;
    • viDoc is configured to emit NMEA-0183 GGA and GST;
    • the device has enough GNSS visibility;
    • logs contain incoming NMEA or satellite messages.
    peripheral.Nmea = message => Debug.Log(message.Raw);

    Without a recent NmeaMessage.Gga, the application cannot create a useful NTRIP task.

    NTRIP Mountpoint Loading Fails

    Use both success and failure callbacks.

    INtripService ntrip = Vigram.NtripService;

ntrip.Mountpoints(
    connection,
    actionSuccess: points => Debug.Log(points.Count),
    actionFailure: error => Debug.LogError(error.Message));

    Check host, port, username, password, HTTPS flags, network availability, and provider access rules. Some providers require HTTPS for mountpoint discovery or correction streams.

    NTRIP Does Not Connect

    Wire NtripState, Start(actionSuccess, actionFailure), and Data.

    ntripTask.NtripState(state => Debug.Log(state.Description()));
ntripTask.Data((bytes, error) =>
{
    if (error != null)
    {
        Debug.LogError(error.Message);
    }
});
ntripTask.Start(
    actionSuccess: () => Debug.Log("Connected"),
    actionFailure: error => Debug.LogError(error.Message));

    Check:

    • the selected mountpoint name;
    • credentials;
    • whether the mountpoint requires NMEA input;
    • whether the initial NmeaMessage.Gga is recent;
    • whether the server returns UNAUTHORIZED, MOUNTPOINT_NOT_FOUND, TIMEOUT, or BAD_RESPONSE.

    Correction Stream Stops

    In 1.0.0, correction stream errors arrive as the second argument of Data.

    ntripTask.Data((bytes, error) =>
{
    if (error != null)
    {
        Debug.LogError(error.Message);
        return;
    }

    Debug.Log($"Correction bytes: {bytes.Length}");
});

    If IGPSService is created with automaticallyReconnectNtrip: true, the SDK attempts reconnect after at least one successful NTRIP connection. Use ShouldDisconnect to update UI and logs.

    GPS Reports A Disconnect Signal

    IGPSService.ShouldDisconnect can report:

    • ConfigurationFailed;
    • DeviceNotFound;
    • NtripDisrupted;
    • InvalidGnssPosition.

    Use the reason to choose a recovery path.

    gps.ShouldDisconnect(signal =>
{
    switch (signal.DisconnectReason)
    {
        case GPSDisconnectSignal.Reason.NtripDisrupted:
            Debug.LogWarning("Show NTRIP reconnect UI.");
            break;

        case GPSDisconnectSignal.Reason.InvalidGnssPosition:
            Debug.LogWarning("Ask the user to move to better GNSS visibility.");
            break;

        default:
            Debug.LogWarning(signal.DisconnectReason);
            break;
    }
});

    Laser Measurement Fails

    Check:

    • the connected device is not DeviceVersion.ViDocLight;
    • CurrentDevice reports the requested laser position as available;
    • the selected LaserConfiguration.Position is valid for the device;
    • duration is finite and within the supported range;
    • actionFailure is logged;
    • the device is not busy with another operation.
    laser.Record(
    configuration,
    actionSuccess: measurement => Debug.Log(measurement.Distance),
    actionFailure: error => Debug.LogError(error.Message));

    For protocol version 1.0, laser duration is clamped to 5-60 seconds.

    Logs

    Pass an ILogger into PeripheralService before startup. See Log A Full Session for the setup pattern.

    If a path is relative, the SDK writes under Application.persistentDataPath. On Android, use the android-documents:// scheme when the app needs to write into the Documents directory.

    Logs are especially useful for support cases involving authentication, scan/connect, peripheral configuration, NMEA availability, NTRIP state, and laser measurement failures.
